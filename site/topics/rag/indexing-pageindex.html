RAG pipeline

    Think of it like...

    Finding a policy in an employee handbook by opening the table of contents vs searching random sticky notes scattered on the floor. Structure beats bag-of-chunks for organized docs.

    PageIndex (TOC Navigation)

    TL;DR: PageIndex builds a TOC-like index and lets an LLM reason over structure to find passages — de-emphasizing embedding similarity over the whole file.

    Why this matters

    PageIndex builds a TOC-like index and lets an LLM reason over structure to find passages — de-emphasizing embedding similarity over the whole file.

    Without this: downstream retrieval and generation inherit upstream mistakes — bigger models and rerankers rarely recover.
    With this: each pipeline stage receives clean, structured input — grounding becomes possible instead of hopeful.

    Play with the machinery

    TOC navigation jumps to the right section in one hop. Flat chunk search may return fragments from wrong places.

    How it works

    Hierarchical index mirrors document outline. LLM chooses branches and drill-down paths before (or instead of) global embedding top-k.

    • Best for: manuals, policies, reports with reliable headings
    • Hybrid: PageIndex routing → small embedding search inside section

    Tradeoffs

    Use when

    • Long structured PDFs with clear section hierarchy
    • Queries referencing sections ("remote work policy")

    Skip when

    • Unstructured notes without headings
    • When pure semantic search over short docs is faster
