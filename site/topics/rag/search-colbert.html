RAG pipeline

    Think of it like...

    Bi-encoder retrieval summarizes a paragraph into one fingerprint. ColBERT keeps every word fingerprint and finds the best word-level matches — like spell-check at retrieval time.

    ColBERT / Late Interaction

    TL;DR: ColBERT scores queries and passages with late interaction (MaxSim) — many token vectors, not one pooled dot product. Strong lexical precision without losing neural signal.

    Why this matters

    ColBERT scores queries and passages with late interaction (MaxSim) — many token vectors, not one pooled dot product. Strong lexical precision without losing neural signal.

    Without this: downstream retrieval and generation inherit upstream mistakes — bigger models and rerankers rarely recover.
    With this: each pipeline stage receives clean, structured input — grounding becomes possible instead of hopeful.

    See the token similarity matrix and how MaxSim sums best matches per query token.

    How it works

    Each query and passage token gets a contextual embedding. Score = sum over query tokens of max similarity to any passage token (MaxSim).

    • Recall stage: often bi-encoder or BM25 first
    • Precision stage: ColBERT rerank on top-k candidates
    • Cost: multi-vector index is larger than single-vector

    Tradeoffs

    Use when

    • Proper nouns, rare terms, short queries
    • As reranker after cheap recall

    Skip when

    • Strict latency/storage budgets with huge corpora
