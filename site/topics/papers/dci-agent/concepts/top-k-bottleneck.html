DCI paper

    Think of it like…

    A librarian who can only hand you 5 pre-selected pages. You can't ask to flip to page 47 or search for an exact product code inside a filing cabinet.

    Top-k Bottleneck

    TL;DR: Retrievers compress corpus access into one ranked list. Evidence dropped at this step never reaches the agent.

    Why this matters

    Agentic search needs exact lexical constraints, sparse clue conjunctions, and hypothesis refinement. A top-k API decides too early what the agent is allowed to see.

    Turn the top-k knob

    Shrink k until ★ gold chunks show LOST — that's the bottleneck the paper describes.

    Mechanism

    Query → embed/score → top-k snippets → agent reasons. If the gold passage ranked #12, the agent never sees it — regardless of model capability.

    Put it together

    This is the problem DCI solves by bypassing the funnel entirely. Next: Interface resolution →
