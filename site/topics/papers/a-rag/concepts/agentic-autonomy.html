A-RAG paper

    Think of it like…

    Following a fixed tour script vs exploring a museum with a map — same building, but only the explorer adapts when a wing looks promising.

    Agentic Autonomy

    TL;DR: True agentic RAG requires autonomous strategy, iterative execution, and interleaved tool use — A-RAG is the only compared method satisfying all three (Appendix A).

    Why this matters

    Workflow RAG (IRCoT, FLARE, MA-RAG) hardcodes retrieval steps at design time. When the model improves, the workflow doesn't — capability gains stall outside the script.

    Paradigm walkthrough

    Step through Naive RAG vs A-RAG — same question, different who decides retrieval.

    Mechanism (§3.3)

    ReAct-style loop: one tool per iteration, observe result, reason, repeat. Deliberately simple — no parallel tool calls — to isolate the effect of hierarchical interfaces on agent behavior.
