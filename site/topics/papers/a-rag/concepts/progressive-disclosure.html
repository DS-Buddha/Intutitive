    Progressive Disclosure

    TL;DR: Search tools return snippets; chunk_read loads full text on demand. A-RAG (Full) uses fewer tokens than Naive RAG despite higher accuracy.

    Why this matters

    A-RAG (Naive) — one embedding tool, no hierarchy — retrieved 56K tokens on MuSiQue vs 5.6K for A-RAG (Full). More tokens, worse accuracy. Dumping context hurts when most chunks are irrelevant.

    Disclosure levels D0–D3

    Toggle from single-shot dump → snippet-only → progressive read → context tracker dedup.

    Mechanism (§3.3)

    Context tracker maintains Cread. Re-reading a chunk returns zero tokens ("This chunk has been read before"), encouraging exploration without context bloat.
